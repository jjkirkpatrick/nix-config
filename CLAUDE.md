## Commands

### Rebuild and switch (system)
```bash
sudo nixos-rebuild switch --flake .#blue-pc
```

### Rebuild and switch (home-manager only)
```bash
home-manager switch --flake .#josh
```

### Using nh (NixOS helper â€” preferred)
```bash
nh os switch .          # system rebuild with diff preview
nh home switch .        # home-manager rebuild with diff preview
nh os boot .            # build for next boot only
nh clean all            # garbage collect old generations
```

### Update flake inputs
```bash
nix flake update
```

### Update a single input
```bash
nix flake update nixpkgs
```

### Check flake for errors
```bash
nix flake check
```

### Format nix files
```bash
nixfmt <file.nix>
# or
alejandra <file.nix>
```

## Architecture

This is a modular NixOS configuration using **Nix Flakes**. There is one host (`blue-pc`) and one user (`josh`).

### Flake structure

`flake.nix` defines two outputs:
- `nixosConfigurations.blue-pc` â€” system-level NixOS config
- `homeConfigurations.josh` â€” user-level Home Manager config

Both are `x86_64-linux`. Unfree packages are allowed globally (needed for NVIDIA, Steam, etc.).

### Module split: core vs home

**`modules/core/`** â€” system-wide NixOS modules (hardware, networking, audio, NVIDIA, Steam, virtualization, Wayland/Hyprland, security, auto-upgrade, garbage collection). All imported via `modules/core/default.nix`.

**`modules/home/`** â€” Home Manager modules for the `josh` user (shell, dotfiles, fonts, dev tools, GUI apps). All imported via `modules/home/default.nix`. Dotfiles are managed declaratively here â€” no manual symlinks needed.

### Host config

`hosts/blue-pc/default.nix` imports `modules/core` and the hardware config. It sets the CPU frequency governor to `performance`.

`hosts/blue-pc/hardware-configuration.nix` is auto-generated by NixOS (from `nixos-generate-config`) and should not be manually edited except for intentional hardware-specific overrides.

### Key inputs

| Input | Purpose |
|---|---|
| `nixpkgs` | unstable channel â€” all packages |
| `home-manager` | user environment management |
| `hyprland` | Wayland compositor |
| `catppuccin` | theme system (applied to both system and home) |
| `ghostty` | terminal emulator (from source) |
| `nur` | Nix User Repository (extra packages) |
| `nix-flatpak` | declarative Flatpak management |

### Adding a new host

1. Create `hosts/<hostname>/default.nix` and `hardware-configuration.nix`
2. Add a new entry under `nixosConfigurations` in `flake.nix`

### Adding packages

- **CLI tools**: `modules/home/packages/cli.nix`
- **GUI apps**: `modules/home/packages/gui.nix`
- **System-wide**: appropriate file in `modules/core/`
- **Dev tools**: `modules/home/development.nix`

### Dotfiles / application config

Hyprland, Waybar, Kitty, Zsh, and other app configs live under `modules/home/config/`. They are written declaratively via Home Manager's `home.file` or program options.
